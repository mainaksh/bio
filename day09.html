Cite PMIDs in [PMID N] format. End with "Confidence: HIGH/MEDIUM/LOW" based on whether the structural neighbors and evidence agree.

Three legs converge into a single grounded surface. Each leg has been built and validated independently over the last four days: /predict Day 5 (XGBoost), /v1/similar Day 7 (HNSW over ChemBERTa embeddings), /v1/evidence Day 8 (BGE over PubMed). Today the composition glue is small — an async asyncio.gather() for parallel fan-out, a jinja2 prompt template, a vLLM HTTP POST, a PMID-validation guard. The complexity is in not adding new functionality but in making the three pre-existing pieces compose cleanly without hallucination or latency regressions.

Why the guard is hard and not soft. RAG-augmented LLMs reduce hallucination but do not eliminate it. The well-documented failure mode: model writes a citation pattern that looks right ("[PMID 12345678]") but references a PMID it pattern-matched from training rather than from the provided evidence. The mitigation is post-hoc validation: extract every PMID from the response, intersect with the input evidence PMIDs, replace unmatched citations with "[citation unavailable for this claim]". This makes the response slightly less fluent in edge cases but regulator-trustable in all cases — the contract becomes: every citation in a /v1/explain response is verifiably one we actually retrieved.

What is NOT in this composition. No multi-turn dialog (V1: chat with the explanation, ask follow-up questions). No streaming response (V1: stream tokens so the reviewer sees the response form in real time; today everything blocks until decode is complete). No model-selection routing (V1: small queries to Llama-3.1-8B, complex queries to Llama-3.1-70B on the V1 GPU node). No human-in-the-loop signoff before serving (V1: reviewer button to flag-and-improve). Today is the simple shape: one query in, one explanation out, every claim sourced.
