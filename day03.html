Why per-dataset recordsize Postgres writes 8K pages; recordsize=16K gives a 2:1 alignment with minimal write amplification. MinIO writes large objects sequentially; recordsize=128K is what Postgres documentation calls "the obvious sweet spot." Redis AOF appends are small; recordsize=8K reduces CoW overhead on each write. ChromaDB writes embedding vectors in larger blobs; 128K. Get this wrong and write amplification can be 4-8× higher than necessary; we'd see it as Day 6+ I/O wait.
