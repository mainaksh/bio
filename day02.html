What we are building today, drawn out

The end-of-day target as a topology diagram. Eight VMs, one bridge, two NUMA nodes, one host-exposed entry point. After 18:00 today this picture is real, not aspirational.

vCPU 72 · RAM 1018 GB · SSD 9.0 TB allocated
NUMA-pinned VM — thick border. tox-inference on node 0, rag-llm on node 1.
Floating VM — thin border. Kernel scheduler decides; NUMA balancing handles cache locality.
br-poc0 bridge — private 10.20.0.0/24, dnsmasq for vm-*.poc.local, default-deny inbound.
Reserve — 220 GB RAM + 4.2 TB SSD held for V1 GPU passthrough and snapshot margin.
NUMA node 0 · cpus 0-35, 72-107
Local memory: 512 GB
  • vm-tox-inference — pinned via cpuset 0-15, memnode 0. PyTorch tensor ops stay in-socket; no cross-NUMA penalty during inference.
  • vm-worker-pool floats; the kernel often schedules it here when the inference VM is idle, which is fine because workers are short-lived.
  • Cross-node distance: 21 vs in-node 10 — ~2.1× access penalty if the kernel scheduler gets it wrong. The pin prevents this.
NUMA node 1 · cpus 36-71, 108-143
Local memory: 512 GB
  • vm-rag-llm — pinned via cpuset 36-47, memnode 1. vLLM kv-cache reads stay local; OMP threads bound to this socket.
  • vm-embed-search floats; the HNSWlib index is small enough (~0.5 GB resident) that NUMA placement does not matter much at our query rate.
  • Verification: numastat -p $(pgrep vllm) after vLLM starts — "node1" hits should be > 95% of total memory references.

The diagram is the contract for today's work. Every line in it — bridge, IP allocation, NUMA pin, ZFS pool, reserve — corresponds to a specific command Jitendra runs in §03. By 17:00 IST today, each rectangle in this picture is a real running process; by 18:00 each has been snapshotted as <name>-base; by 20:00 the host has survived a 10-minute 72-core hot-soak and we know its thermal envelope.

Three things that are not in this picture are deliberate omissions worth naming. First, there are no Postgres / MinIO / Redis / ChromaDB processes inside vm-data-store yet — today only the empty Ubuntu container exists; Day 3 fills it. Second, there is no Traefik / FastAPI inside vm-core-api — that's Day 5. Third, there is no Prometheus / Grafana inside vm-observability — that's Day 5 as well. Today provisions the rooms; the next three days move the furniture in.

11:30–13:00 Canonicalization round-trip — idempotency proof on a 100-compound sample from each.

The fundamental assumption of the curation pipeline is that canonicalize(canonicalize(x)) == canonicalize(x) — running RDKit's canonicalization twice produces the same string the second time as the first. If that assumption fails for even a small percentage of compounds, the entire pipeline has an edge case we did not budget for.

  1. Round-trip script running on a stratified 100-compound sample from each dataset.
(aitip-base) sarthak@laptop:~/aitip-data$ cat > round_trip_check.py <<'PYEOF' """Canonicalization idempotency check.
